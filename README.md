# Дашборд «Склад · Продажи»

Витрина остатков и продаж по 4 основным складам поверх вашей базы PostgreSQL
`Ostatki` (таблицы `Остатки` и `Продажи`). Бэкенд — FastAPI, считает все метрики
на стороне Python; фронт — один HTML, тянет данные постранично (держит тысячи
товаров без тормозов).

## Что показывает
- KPI по 4 складам: выручка + тренд последнего месяца.
- Таблица в стиле Excel: остатки (**Сейчас**: в наличии / отгружается / в резерве /
  доступно; **Ожидается**: приход / расход / остаток), продажи, дни запаса,
  рекомендуемый запас, Δ заказ, метки, ABC/XYZ, ликвидность.
- Метки: 🔥 **Hot** (растущий спрос), **Дозаказ** (уйдёт в ноль), **Залежь**
  (оверсток), **Неликвид** (не продаётся).
- Клик по товару → отдельная страница с большими графиками: продажи по месяцам
  с разбивкой по складам (один артикул сразу на всех 4 складах) + выручка по
  складам + таблица остаток/реком/Δ.

## 1. Установка (один раз)

Нужен Python 3.10+ и запущенный PostgreSQL с вашей базой.

```bash
cd sklad
pip install -r requirements.txt
```

## 2. Настройка подключения

Откройте `app.py`, блок `DB` вверху — впишите свои данные (по умолчанию уже
стоят `Ostatki / postgres / 123 / localhost:5432`):

```python
DB = dict(dbname="Ostatki", user="postgres", password="123",
          host="localhost", port="5432")
```

Там же `MAIN_WAREHOUSES` — четыре склада и их отображаемые имена. Имена слева
должны точно совпадать со значениями колонки `"Склад"` в базе (нормализованные,
напр. `алматы_основной_склад`). Если не уверены в написании — выполните в базе:

```sql
SELECT DISTINCT "Склад" FROM "Продажи" ORDER BY 1;
```

и подставьте точные строки.

## 3. Запуск

```bash
python -m uvicorn app:app --host 0.0.0.0 --port 8000
```

Откройте **http://localhost:8000**

При старте в консоли появится `[OK] витрина загружена: N артикулов…`.
Если вместо этого `[ОШИБКА подключения к БД]` — проверьте параметры `DB` и что
PostgreSQL запущен.

## 4. Показать другим по сети

**В локальной сети (тот же Wi-Fi / офис):**
Сервер уже слушает `0.0.0.0`. Узнайте IP своего ПК (`ipconfig` в Windows →
строка IPv4, напр. `192.168.1.50`) и дайте коллегам ссылку:

```
http://192.168.1.50:8000
```

Если не открывается — разрешите порт 8000 в брандмауэре Windows (Защитник
Windows → Дополнительные параметры → Правила для входящих → создать правило для
порта 8000 TCP).

**Внешняя ссылка (из любого места, без настройки сети):**
Поднимите туннель к локальному серверу. Самый простой — Cloudflare Tunnel:

```bash
cloudflared tunnel --url http://localhost:8000
```

Он выдаст публичный `https://…trycloudflare.com` адрес. Аналогично работает
`ngrok http 8000`. Это для демо; для постоянной работы лучше развернуть на
сервере/VPS.

## 5. Обновление данных (раз в месяц)

После того как вы загрузили в базу новый месяц продаж/остатков своими
Python-скриптами — нажмите кнопку **⟳ Обновить** в интерфейсе (или откройте
`http://localhost:8000/api/refresh`). Витрина перечитает базу без перезапуска
сервера.

---

## 6. Как устроено (и куда расти)

Слои разделены: **база** → `metrics.py` (расчёт) → `app.py` (API) → `index.html`
(отображение). Менять один слой можно не трогая остальные.

- **Подключить ML-прогноз** — замените тело функции `forecast_next_month()` в
  `metrics.py` на предсказание модели. Остальное (рекомендация запаса, метки,
  витрина) не меняется.
- **Добавить срок годности** — заведите колонку срока в `Остатки`, и в
  `compute_row_metrics` ограничьте `recommended` величиной `vel_day × срок`,
  чтобы не рекомендовать оверсток скоропорта.
- **Добавить склад** — допишите строку в `MAIN_WAREHOUSES`. Появится вкладка и
  колонка, запросы не переписываются.
- **Параметры запаса** (срок поставки, покрытие, уровень сервиса) — константы
  `LEAD_DAYS`, `COVER_MONTHS`, `SERVICE` вверху `metrics.py`.

## API (для справки)
- `GET /api/kpis` — карточки складов + список месяцев.
- `GET /api/table?view=summary|<склад>&page=&size=&sort=&direction=&q=` —
  таблица с пагинацией/сортировкой/поиском.
- `GET /api/product/{артикул}` — данные для детальной страницы.
- `GET /api/refresh` — перечитать базу.
